# ETTh1 결측 시계열 예측

결측치가 포함된 ETTh1_adjust 데이터에 대한 다변량 시계열 예측 과제.
입력 96시간 → 예측 96시간(long-term forecast).

---

# 1. 데이터 분석 및 전처리

# 1-1. 데이터 분석
- 통계량(describe), boxplot, IQR 이상치, 정상성(ADF), 시계열 분해(decompose) 수행
- 결측 형태: NaN이 아니라 시간 행 자체가 삭제됨. `date.diff()`로 573군데의 시간 끊김 확인(간격 6~19시간, 불규칙)
- 정상성: 전 변수 ADF 검정. OT는 p=0.0056으로 정상. (EDA 목적이며 모델 선택의 직접 근거는 아님)
- 분해: 24시간 주기 기준 분해에서 trend 성분이 크고 seasonal은 상대적으로 작음. (24h 기준 관찰이며 "계절성 없음"으로 단정하지 않음)
- 이상치: IQR상 존재. 특히 OT는 이상치 395개가 전부 상한(하한 0)

# 1-2. 전처리 결정
- 결측 처리(하이브리드): 8시간 이하 gap은 선형 보간, 8시간 초과 gap은 window 경계로 제외
  - 처음엔 보간하지 않으려 했으나, gap 걸친 window 제외 규칙을 엄격히 적용하면 최대 연속 구간(153행)이 window 크기(192행)에 미달해 학습 데이터가 0개가 됨
  - threshold별 window 수를 실험(6h 부족 / 8h 5911 / 10h 9409)한 뒤, 보간 신뢰성과 학습 표본 수의 균형점으로 8시간 선택
  - 보간은 df를 1시간 등간격으로 만들기 위함이고, 8시간 초과 구간은 값을 채워도 window에서 제외해 인위적 보간값의 학습 유입을 이중으로 차단
- 이상치: 제거하지 않음. OT 이상치가 전부 상한에 편중된 것은 대칭 노이즈가 아니라 실제 전력 수요 피크일 가능성이 높다고 판단. 제거 시 타겟의 피크 정보를 손실. (Informer 구현에서도 별도 이상치 제거를 수행하지 않음)
- 보간 방식: 선형. 채우는 gap이 짧고, 곡선/스플라인은 변동 큰 전력 데이터에서 없던 극값(오버슈팅)을 만들 위험이 있어 가장 보수적인 선형을 채택
- 정규화: StandardScaler를 train 구간에만 fit 후 전체 transform. 데이터 누수 방지(특히 시계열은 test가 미래 데이터라 전체 fit 시 미래 정보 유입)

# 1-3. 전처리 후 재분석
- describe: 원본과 거의 동일(OT 평균 13.3, std 8.55) → 보간이 분포를 왜곡하지 않음
- ADF: p=0.0082, 여전히 정상 → 정상성 유지
- decompose: 24시간 기준에서 여전히 trend가 크고 seasonal이 작음

# 2. 데이터셋 및 로더 (`create_dataloader_default`)

- config의 argument(window_size, pred_len, scaler, split_rate, timeenc, freq 등)를 모두 반영
- split: train/valid/test = 7:1:2. valid/test 시작을 window_size만큼 앞당겨 각 구간 첫 시점 예측에 필요한 입력 문맥을 확보(입력으로만 사용, 정답 아님)
- gap 제외(요구사항 1-7): 8시간 초과 gap 구간을 boundary로 표시하고, window가 이를 밟으면 시작점 후보에서 제외
- 시간 피처: timeenc=0에 따라 month/day/weekday/hour 생성(모델 인터페이스 호환용)

# 3. 모델

동일 전처리·학습 설정에서 modelname만 변경해 5개 모델을 비교했다.

| 모델 | MSE | MAE | 인접차(예측 매끄러움) | 특징 |
|---|---|---|---|---|
| DLinear(공유) | 0.596 | 0.554 | 0.115 | 분해 + 공유 Linear |
| DLinear(개별) | 0.754 | 0.640 | 0.359 | 분해 + 개별 Linear, 진동 폭발 |
| NLinear | 0.859 | 0.660 | (심함) | 마지막값 의존, 최저 성능 |
| MLP | 0.637 | 0.569 | 0.055 | 비선형 통짜, 진동 없음 |
| PatchTST | 0.508 | 0.502 | 0.033 | 패칭 + 어텐션, 최종 선택 |

# 실험 과정 (모델 선택 근거)
1. DLinear로 시작 → 예측이 인접 시점마다 과도하게 진동(인접차 0.115 vs 실제 0.051)
2. 원인 추적(실험 기반 해석):
   - 데이터 문제? → 같은 데이터의 MLP는 매끄러움(0.055) → 데이터 요인은 아닌 것으로 판단
   - 24시간 기준에서 seasonal 성분이 작게 관찰된 상황에서, DLinear의 seasonal 성분 예측 과정이 진동에 영향을 준 것을 가능한 원인으로 해석
   - DLinear individual=True(자유도 증가) → 진동 폭발(0.359) → 위 해석을 뒷받침(과적합 경향)
3. MLP(분해 없는 비선형) → 진동 해결(0.055), 성능은 중간
4. PatchTST → 성능 최고(0.508) + 진동 없음(0.033)

# PatchTST를 최종 모델로 선택한 이유
- 패칭: 96시간을 16시간 단위 patch로 나누어 트랜스포머가 patch 단위 특징을 학습하도록 구성. 실험에서는 DLinear보다 더 매끄러운 예측을 보임
- 채널 독립 + 인스턴스 정규화: 변수를 각각 독립 시퀀스로 처리하고 구간별로 정규화 → 분포 변화에 강건
- 어텐션: 어텐션을 통해 패치 간 관계(장거리 의존성)를 학습할 수 있으며, 실험에서는 DLinear보다 더 안정적인 예측을 보임

# 4. 결과 및 해석

# 4-1. 지표
- MSE와 MAE를 함께 사용. MSE는 큰 오차에 더 큰 페널티를 주고, MAE는 평균적인 오차 크기를 보여줘 두 관점을 함께 평가
- 예측 매끄러움 지표(인접차): 인접 시점 차이의 평균(`mean(|pred[t]-pred[t-1]|)`). 예측이 과도하게 진동하는지 정량화. PatchTST는 0.033으로 실제값(0.051)만큼 매끄러움
- PatchTST MSE 0.508은 원본 ETTh1 벤치마크(약 0.38)보다 높으나, 결측이 포함된 adjust 데이터 + pred_len 96 조건이라 직접 비교는 부적절

# 4-2. 시각화
- 실제값(True) vs 예측값(Pred)을 여러 구간에 대해 비교(4-1 셀의 plot)

# 4-3. 한계 및 개선 방향
한계
- 큰 변동(봉우리/골짜기) 과소예측: 예측 표준편차가 실제보다 작음(모든 모델 공통)
- gap 경계 오염: boundary는 gap을 직접 밟는 window만 제외하므로, gap 직후 시작하는 window는 보간값의 영향이 일부 잔존(학습량 확보와의 타협)
- 하이퍼파라미터 미튜닝: patch_len, lr, epoch 등은 기본값 사용
- 변수별 성능 미분석: 전체 평균 MSE만 확인
- 계절성은 24시간 기준만 확인(주간 168시간 등 미검증)

개선 방향
- window 확대(96→336)로 더 긴 문맥 확보
- 피크 구간 가중 손실로 OT 상한 이상치(실제 신호) 예측 강화
- 하이퍼파라미터 튜닝(grid search)
- 생성만 하고 미사용 중인 시간 피처(시간대/요일) 활용
